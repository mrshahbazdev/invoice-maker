# 中核となるビジネス設定とブランディング

最初の請求書を作成する前に、コア ビジネス プロファイルを適切に構成することが絶対に必要です。このプロファイルは、文書の法的な表示と、クライアントがあなたのブランドをどのように視覚的に認識するかを大きく左右します。

**[設定] > [ビジネス設定]** に移動して、マスター構成ターミナルにアクセスします。

---

## 1. 法的表示とコンプライアンス
請求書発行は単にお金を要求するだけではありません。これは法的な取引であり、管轄区域の課税規制を厳密に遵守する必要があります。

### ビジネス情報
これらのフィールドは、生成されたすべての PDF ドキュメントに永続的に表示されるため、完全に入力されていることを確認してください。
- **法人名:** 貴社の登録された完全な法人名 (例: 「Acme Corp LLC」)。
- **登録住所:** 本社の物理的な所在地。
- **納税者番号 (TIN/VAT):** これは、プラットフォーム上で最も重要なフィールドの 1 つです。お住まいの国に応じて (例: 米国では EIN が必要、ヨーロッパ諸国では​​ VAT 番号が必要)、ここに課税 ID を入力する必要があります。保存すると、この ID 文字列が作成するすべての請求書の法的ヘッダーに自動的に挿入され、シームレスに最大限の法的コンプライアンスが確保されます。

---

## 2. 高度なグローバル テーマとビジュアル
このプラットフォームを使用すると、確立されたブランド アイデンティティに合わせて送信ドキュメントだけでなく、ソフトウェア インターフェイス自体も完全に再スキンすることができます。

### ビジュアルアセットのアップロード
- **企業ロゴ:** 高解像度の PNG または JPEG をアップロードします。システムは、送信される請求書とカスタマー ポータル内の両方で、このロゴのサイズと位置を自動的に変更します。
- **ファビコン:** 小さな `.ico` または `.png` ファイルをアップロードします。このアイコンは、クライアントがパーソナライズされたポータルにログインするときに Web ブラウザーのタブに表示され、完全にホワイトラベルのエクスペリエンスが保証されます。

### アルゴリズムによる SaaS テーマ
管理者として操作している場合は、固有の **テーマ** タブに移動できます。
- ここでは、会社のコアとなる 16 進数の原色 (例: `#FF5733`) を入力します。
- **AI カラー エンジン:** この色を保存すると、当社の AI が詳細に分析します。次に、15 シェード全体の調和のとれたカラー パレット (明暗の完全なコントラスト) を動的に生成し、それを SaaS インターフェイス全体に適用します。サイドバー、ボタン、プログレスバー、アラートはすべて、暗黙のうちにブランドに一致するように瞬時に変換されます。

## 3. 即時出力のための Stripe の統合
迅速に支払いを受け取ることがプラットフォームの最終目標です。銀行インフラの統合には 30 秒かかります。

- グローバル設定内の [**お支払い**] タブに移動します。
- **「Stripe で接続」** をクリックします。 
- Stripe のオンボーディング ポータルに安全にルーティングされます。すでに Stripe アカウントをお持ちの場合は、すぐにログインできます。そうでない場合でも、3 分で作成できます。
- **マジック リンク:** 相互接続されると、この時点から生成するすべての請求書は、プログラムによってクライアント ポータル内に巨大な「今すぐ支払う」ボタンを生成します。顧客がボタンをクリックし、安全性の高いモーダルにクレジット カードを入力すると、お金が企業の銀行口座に直接振り込まれます。さらに、プラットフォームは手動でデータを入力しなくても、社内の請求書に自動的に「支払い済み」のフラグを立てます。
